# 固定资产质检流程与 SOP

本文沉淀资料库中固定资产标准底稿、SOP 和程序执行方法的核心流程，供固定资产质检 Agent 在设计规则、提示用户补充资料、生成报告时引用。

资料来源：

- `固定资产质检agent/资料库/FY26_SOP K1 SWP 固定资产.xlsx`
- `固定资产质检agent/资料库/K1 SWP 固定资产 202YMMDD XYZ公司.xlsx`
- `固定资产质检agent/资料库/K.03.1 SAP-中精确度.xlsx`

## 适用范围

- 适用于固定资产科目被评估为重大账户时的结构化台账质检和底稿完整性检查。
- 当前只沉淀 SWP/Excel 底稿内的信息，不替代审计准则、GAM 或项目组职业判断。
- 如果固定资产被评估为有限风险账户，应另行参考有限风险账户底稿和项目风险评估结论。
- 不包含固定资产减值、盘点、权属检查等 OSP 程序；这些程序后续可单独扩展。

## 总体流程

```text
确认底稿范围与基础信息
        |
        v
K.00 Lead Sheet：两期变动、预期分析、异常波动调查
        |
        v
K.01 Agree SL to GL：后推明细表与总账/明细账/资产清单核对
        |
        v
K.02 Additions/Disposals：新增与处置清单、样本总体、详细测试
        |
        v
K.03 Depreciation：SAP/TOD 折旧测试与折旧政策复核
        |
        v
汇总执行结论、异常事项、拒绝执行原因和人工复核项
```

## 汇总页

汇总页用于记录固定资产相关 PSP 是否执行，以及不执行时的理由。

Agent 质检关注点：

- 是否按项目风险情况选择执行或不执行各程序。
- 不执行新增、处置、折旧测试时，是否有充分理由。
- 不能仅因本期折旧计提金额小于 TE 就直接判断无需执行折旧测试。
- 当新增原值、处置净值、折旧计提发生额小于 TE 但大于 TT 时，应考虑风险评估、总体特征和是否需要补充判断性抽样。
- 若满足有限风险账户条件并被评估为有限风险账户，应使用对应 LRA 底稿，而非直接跳过程序。

## K.00 Lead Sheet

目标：

- 获取和编制固定资产两期变动引导表。
- 将本期余额与试算平衡表核对一致。
- 将上期余额与上年审定数核对一致。
- 对固定资产变动建立预期，并调查不符合预期或异常的变动。

关键输入：

- 客户名称。
- 期末日期。
- 分析日期。
- 可容忍误差 `TE`。
- 名义金额 `SAD`。
- 适用会计准则。
- 记账本位币与单位。

Agent 质检关注点：

- `TE`、`SAD` 是否与 Canvas 或项目最终设定一致。
- 审定阶段与执行阶段 `TE`、`SAD` 是否发生变化；如变化，是否评估审计范围和程序影响。
- 是否记录预期分析和异常波动调查。
- 是否将调整分录或账表调整与后续程序保持一致。

## K.01 Agree SL to GL

目标：

- 获取上期末至本期末固定资产及相关新增、处置、转让、折旧的后推明细表。
- 将后推明细表与总账、明细账、固定资产清单、试算表核对一致。
- 调查超过 `SAD` 的差异。

后推明细表常见来源：

- 公司自行编制的固定资产后推表。
- 固定资产台账、总账、试算表、JE 汇总后自行编制。
- 久其等系统报表或固定资产附注。

Agent 质检关注点：

- 原值、累计折旧、减值准备、净值的期初、本期增加、本期减少、期末余额是否逻辑一致。
- 本期减少应以负数或明确减少列呈现，并与底稿口径一致。
- 处置或报废时，是否存在累计折旧大于原值等异常。
- 后推明细表、资产清单和试算表差异超过 `SAD` 时是否调查。
- 上年调整是否正确跟调。

## FA List

目标：

- 获取期末固定资产明细清单。
- 将固定资产清单金额与 K.01 后推明细表核对。
- 为折旧重新计算、SAP 或其他细节测试提供基础数据。

必需字段：

- 固定资产类别。
- 固定资产编号。
- 固定资产名称。
- 入账开始日期。
- 使用寿命（月）。
- 残值率。
- 原值。
- 累计折旧。
- 减值准备。
- 净值。

Agent 质检关注点：

- 资产清单不要求强行重分类为标准类别，应保留被审计单位台账中的类别。
- 必需字段缺失时，应返回 `FAIL` 或 `NEED_REVIEW`。
- 如果折旧测试无法通过 SAP 获得充分证据，可在 FA List 中执行全量 by item 折旧测算。

## K.02.1 新增测试

目标：

- 获取当期按类别划分的固定资产新增清单。
- 将新增清单与后推明细表核对一致。
- 调查超过 `SAD` 的差异。
- 判断新增测试总体是否具备共同特征。

新增清单必需字段：

- 固定资产类别。
- 固定资产编号。
- 固定资产名称。
- 入账开始日期。
- 使用寿命（月）。
- 残值率。
- 原值。
- 新增方式。

Agent 质检关注点：

- 新增测试样本总体通常为购置新增原值。
- 在建工程转入、企业合并、其他转入等新增方式不一定与购置新增具备相同风险特征，应单独考虑程序。
- 新增清单与后推明细表差异超过 `SAD` 时应调查。
- 新增测试 TOD 证据描述应关注控制权转移时点证据，不能只记录合同和发票。

## K.02.2 处置测试

目标：

- 获取本期按资产类别划分的固定资产处置清单。
- 将处置清单与后推明细表核对一致。
- 调查超过 `SAD` 的差异。
- 评估处置记录完整性。

处置清单必需字段：

- 固定资产类别。
- 固定资产编号。
- 固定资产名称。
- 原值。
- 累计折旧。
- 减值准备。
- 净值。
- 处置日期。
- 减少方式或处置/报废方式。

Agent 质检关注点：

- 处置测试总体通常为出售和报废导致减少的资产净值。
- 其他减少方式可能不具备相同风险特征，应单独考虑程序。
- 资产处置损益不是本底稿处置测试的样本总体，应在相关损益底稿中测试。
- 应通过询问、董事会决议、收入账户分析等方式关注是否存在未记录处置。
- 结合 K.00 预期分析评估处置完整性。

## K.03.1 折旧 SAP

目标：

- 通过实质性分析程序验证固定资产和利润表折旧费用金额是否合理列示。
- 在适当 CRA 和实体类型下，以 SAP 获取充分、适当的审计证据。

SAP 类型：

- 中精确度：通常用于 `CRA = Minimal` 的场景。
- 高精确度：通常用于 `CRA = Low/Moderate/High` 的场景。

Agent 质检关注点：

- 执行折旧测试前，应先选择合适测试方法。
- `CRA` 应与 K.00 Lead Sheet 中计价/计量认定保持一致。
- 实体类型选择应与抽样或非复杂方法判断一致。
- 当识别折旧费用存在特别风险，且不依赖控制或控制无效时，应执行 TOD。
- SAP 证据不足时，应补充详细测试。

## K.03.2 折旧 TOD / By Item 测试

目标：

- 获取固定资产本期计提折旧清单。
- 将折旧计提清单与后推明细表核对一致。
- 按公司折旧政策重新计算本期应计提折旧，并与账面折旧比较。

典型计算逻辑：

```text
本期应计提折旧 = 原值 * (1 - 残值率) / 使用寿命（月） * 计提月份或权重
差异 = 重新计算折旧 - 账面折旧
```

Agent 质检关注点：

- 折旧测试总体应与本期计提折旧金额一致，并与后推明细表核对。
- 单项资产差异或合计差异大于 `SAD` 时，应进一步核查。
- 当资产数量较少、新增/处置较多、提足折旧时点不集中、存在减值准备时，by item 可能更有效。
- 涉及减值、后续资本性支出、使用寿命重新估计时，标准公式可能不适用，应进入人工复核。

## K.03.3 折旧政策复核

目标：

- 评估固定资产折旧政策合理性。
- 检查折旧方法、使用寿命、预计净残值是否清晰且符合准则。
- 检查本期政策与上期是否一致。
- 若政策变更，检查依据是否充分。

Agent 质检关注点：

- 折旧政策至少包括折旧方法、使用寿命、预计净残值。
- 使用寿命、预计净残值、折旧方法变化通常作为会计估计变更，采用未来适用法。
- 企业至少应于每年年度终了复核使用寿命和预计净残值。
- 折旧政策发生变化但缺少依据时，应返回 `NEED_REVIEW` 或 `FAIL`，具体取决于团队规则。

## Agent 输出建议

当 Agent 发现问题时，建议输出：

- 涉及程序：如 `K.01 Agree SL to GL`。
- 涉及对象：如 FA List、新增清单、处置清单、折旧测试。
- 问题类型：字段缺失、金额不一致、程序未执行、证据不足、需人工判断。
- 严重级别：`PASS`、`WARN`、`FAIL`、`NEED_REVIEW`。
- 建议动作：补充字段、核对后推表、说明拒绝执行理由、补充 TOD 或人工复核。
