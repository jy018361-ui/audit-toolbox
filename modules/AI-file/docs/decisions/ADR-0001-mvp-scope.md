# ADR-0001：MVP 范围

## 状态

已采纳。

## 背景

固定资产质检 Agent 后续可能涉及台账、影像、合同、发票、ERP 接口和人工复核流程。为了尽快形成可验证闭环，第一阶段需要控制范围。

## 决策

MVP 只处理脱敏后的结构化固定资产台账数据，优先完成以下能力：

- 读取 Excel/CSV 或测试 fixture。
- 标准化固定资产字段。
- 执行基础质检规则。
- 输出结构化质检问题。
- 使用自动化测试保障规则行为。

## 首批规则

- 必填字段校验。
- 资产编码唯一性校验。
- 原值、累计折旧、净值金额关系校验。
- 金额非负校验。
- 启用日期合理性校验。

## 暂不纳入 MVP

- 真实生产系统接入。
- OCR、影像识别和附件比对。
- 数据库持久化。
- 审批流和人工复核页面。
- 大规模性能优化。

## 影响

- 团队可以先围绕 `src/ingest/`、`src/rules/`、`src/report/` 并行开发。
- 所有样例数据必须脱敏并放入 `tests/fixtures/`。
- 后续扩展影像、API 和审批能力时，需要新增 ADR。
