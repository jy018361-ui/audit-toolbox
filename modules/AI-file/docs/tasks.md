# 任务清单

本文记录固定资产质检 Agent 的阶段性任务。适合放待办事项、负责人和验收标准；每日接续进展仍以 `docs/handoff/latest.md` 为准。

## MVP 任务

| 状态 | 任务 | 负责人 | 验收标准 |
| --- | --- | --- | --- |
| 待开始 | 准备脱敏固定资产样例数据 | 待定 | `tests/fixtures/` 中存在基础样例和异常样例 |
| 待开始 | 设计标准底稿字段映射 | 待定 | 覆盖 `FA list`、`新增清单`、`处置清单`、`K.03.2 折旧测试TOD` |
| 待开始 | 初始化 Python 工程配置 | 待定 | 可运行测试命令，依赖记录清楚 |
| 待开始 | 实现 `fa_list_required_fields` | 待定 | 测试覆盖通过和字段缺失场景 |
| 待开始 | 实现资产编码唯一性校验 | 待定 | 重复编码返回 `FAIL` |
| 待开始 | 实现金额关系校验 | 待定 | 原值、累计折旧、减值准备、净值关系异常返回 `FAIL` |
| 待开始 | 实现使用寿命和残值率校验 | 待定 | 使用寿命非正、残值率超范围返回 `FAIL` |
| 待开始 | 实现新增/处置/折旧清单字段完整性校验 | 待定 | 三类清单缺少必需字段时返回 `FAIL` |
| 待开始 | 实现基础报告汇总 | 待定 | 可输出资产级结论和问题明细 |

## 任务维护规则

- 新增任务时必须写清验收标准。
- 任务完成后更新状态，并在 `docs/handoff/latest.md` 写明关键结果。
- 如果任务改变架构或规则口径，补充 `docs/decisions/` 或 `docs/domain-glossary.md`。
